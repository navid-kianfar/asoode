# Work Packages

## Overview
Work packages are the boards within a project where actual task management happens. Think of them as Kanban boards that hold lists and tasks.

## Creating a Work Package
1. Open a project
2. Click "New Work Package"
3. Choose a template:
   - **Blank**: Empty board
   - **Kanban**: To Do, In Progress, Review, Done
   - **Weekday**: Monday through Friday
   - **Team Members**: One column per member

## Board Features
- **Drag-and-drop**: Move tasks between lists
- **Custom lists**: Create as many lists as needed
- **Labels**: Color-coded tags for categorization
- **Sorting**: Sort tasks by various criteria
- **Filtering**: Filter by member, label, or date
- **Permissions**: Control who can view and edit

## Views
- **Board view**: Traditional Kanban layout
- **List view**: Flat list of all tasks
- **Calendar view**: Tasks organized by due date
- **Timeline view**: Gantt-style timeline

## Best Practices
- Keep lists focused (5-7 tasks per list is ideal)
- Use labels consistently across work packages
- Archive completed tasks regularly
- Review board progress in weekly standups
