# Kanban Boards Guide

## What is Kanban?
Kanban is a visual project management method that helps teams visualize work, limit work-in-progress, and maximize flow. In Asoode, Kanban boards are the primary way to manage tasks within work packages.

## Setting Up Your Board
1. Create a work package within your project
2. Choose the "Kanban" template or start blank
3. Customize your lists to match your workflow

## Common Kanban Workflows

### Software Development
- **Backlog** → **To Do** → **In Progress** → **Code Review** → **Testing** → **Done**

### Marketing
- **Ideas** → **Planning** → **In Progress** → **Review** → **Published**

### General
- **To Do** → **In Progress** → **Done**

## Managing Lists
- **Add a list**: Click "Add List" at the right edge of the board
- **Rename**: Click on the list title to edit
- **Reorder**: Drag lists by their header
- **Archive**: Remove completed lists

## Moving Tasks
- **Drag and drop**: Click and drag a task card to another list
- **Quick move**: Use the task menu to move to a specific list
- **Bulk move**: Select multiple tasks and move together

## Labels and Colors
- Create color-coded labels for categorization
- Apply multiple labels to a single task
- Filter the board by label to focus on specific types of work

## Best Practices
1. **Limit WIP**: Don't let too many tasks pile up in "In Progress"
2. **Daily standups**: Review the board with your team daily
3. **Regular grooming**: Archive done tasks, review backlog weekly
4. **Clear definitions**: Define what "Done" means for your team
5. **Consistent labels**: Use the same label system across boards
