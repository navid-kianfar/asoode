# Access Control Guide

## Overview
Asoode provides granular access control at every level — from groups and projects to individual work packages. Understanding access levels helps you keep your data secure while enabling collaboration.

## Access Levels

### Owner
- Full control over the resource
- Can delete the resource
- Can manage all settings and members
- Only one owner per resource

### Admin
- Can manage members and settings
- Can create and edit content
- Cannot delete the resource
- Cannot change the owner

### Editor
- Can create and edit tasks, lists, and content
- Cannot manage members or settings
- Visible in member lists

### Hidden Editor
- Same permissions as Editor
- Not visible in member lists
- Useful for support staff or external collaborators

### Visitor
- View-only access
- Cannot create or edit content
- Can view tasks, boards, and files

## Access Inheritance
- Group access propagates to projects within the group
- Project access propagates to work packages within the project
- You can override inherited access at any level

## Managing Access
1. Navigate to the group, project, or work package settings
2. Click "Members" or "Access"
3. Add members or change existing access levels
4. Changes take effect immediately

## Best Practices
- Follow the principle of least privilege — give the minimum access needed
- Use groups to manage access at scale rather than per-project
- Review access levels regularly, especially when team members change roles
- Use Hidden Editor for external contractors who don't need to appear in team lists
- Document your access control policies for your team
